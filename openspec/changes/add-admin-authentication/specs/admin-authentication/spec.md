## ADDED Requirements

### Requirement: Admin 后台页面必须要求已认证会话
系统 SHALL 在进入 admin 后台页面前验证当前请求是否具备有效的管理员会话；未认证用户 MUST 被引导到登录页面，而不是直接渲染后台内容。

#### Scenario: 未登录访问后台首页
- **WHEN** 用户在没有有效管理员会话的情况下访问 `/` 或任一 admin 前端路由
- **THEN** 系统跳转或返回登录页面，且不展示后台内容

#### Scenario: 已登录访问后台
- **WHEN** 用户携带有效管理员会话访问 admin 前端路由
- **THEN** 系统返回对应后台页面并允许后续管理操作

### Requirement: 系统必须提供管理员登录会话接口
系统 SHALL 提供管理员登录、会话查询和登出接口，用于建立、探测和清除后台登录态。

#### Scenario: 登录成功
- **WHEN** 客户端向登录接口提交正确的管理员用户名与密码
- **THEN** 系统创建有效会话并通过安全 Cookie 返回给客户端

#### Scenario: 登录失败
- **WHEN** 客户端向登录接口提交错误的管理员用户名或密码
- **THEN** 系统返回认证失败响应，且不创建会话

#### Scenario: 查询当前会话
- **WHEN** 客户端请求管理员会话状态接口
- **THEN** 系统返回当前请求是否已认证，以及会话对应的管理员身份摘要

#### Scenario: 登出
- **WHEN** 已登录客户端请求登出接口
- **THEN** 系统清除现有管理员会话，使后续后台访问重新要求登录

### Requirement: 管理 API 必须统一拒绝未认证请求
系统 SHALL 对 admin 使用的管理 API 执行统一鉴权校验；任何未携带有效管理员会话的请求 MUST 返回 401，且不得返回受保护数据。

#### Scenario: 未认证读取管理数据
- **WHEN** 客户端在未登录状态下请求任一受保护管理 API
- **THEN** 系统返回 401，且响应中不包含目标管理数据

#### Scenario: 未认证修改管理数据
- **WHEN** 客户端在未登录状态下向任一受保护管理 API 发送写操作请求
- **THEN** 系统返回 401，且不执行对应写入或删除动作

#### Scenario: 已认证访问管理 API
- **WHEN** 客户端携带有效管理员会话请求受保护管理 API
- **THEN** 系统按原有业务语义处理请求并返回正常结果

### Requirement: Admin 鉴权配置必须显式且安全
系统 SHALL 通过显式配置启用 admin 鉴权，并要求管理员密码以哈希形式配置，同时要求提供会话签名密钥；系统 MUST 拒绝使用不完整或不安全的默认配置启动受保护后台。

#### Scenario: 配置完整时启动后台
- **WHEN** 部署配置提供了启用鉴权所需的管理员用户名、密码哈希和会话密钥
- **THEN** 系统正常启动并以鉴权模式暴露 admin 页面与管理 API

#### Scenario: 缺少关键配置
- **WHEN** 系统计划暴露 admin 页面或管理 API，但缺少管理员用户名、密码哈希或会话密钥中的任一项
- **THEN** 系统拒绝以不安全配置启动受保护后台，并报告配置错误
