## Why

当前 admin 后台和其 `/api/*` 管理接口默认暴露，无需登录即可查看会话、配置、技能、日志等敏感数据，部署配置中还包含高敏感凭据。随着系统进入持续运行和远程部署阶段，后台入口必须具备最基本的访问控制能力，避免管理面被未授权访问。

## What Changes

- 为 admin 后台增加登录态，未认证用户不能访问管理页面。
- 为管理 API 增加统一鉴权校验，未认证请求返回 401，不再直接暴露管理数据。
- 提供受控的登录、登出和鉴权状态查询接口，供 admin 前端维持会话。
- 增加 admin 鉴权配置项与部署约束，要求显式配置管理员凭据或等价密钥后才能启用后台访问。
- **BREAKING**：未登录访问 admin 页面或管理 API 的行为将从“直接成功”改为“被拒绝并要求认证”。

## Capabilities

### New Capabilities
- `admin-authentication`: 为 admin 前端和管理 API 提供统一的登录鉴权、会话保持与未认证拦截能力。

### Modified Capabilities

## Impact

- `agent/internal/api/*`：需要增加鉴权中间层并保护现有管理接口。
- admin 前端路由与 API 客户端：需要支持登录页、会话检测、401 处理与登出。
- Agent 启动配置与部署配置：需要新增管理员鉴权相关配置和上线要求。
- 运维行为：部署时需要显式提供 admin 凭据，避免后台默认裸奔。
