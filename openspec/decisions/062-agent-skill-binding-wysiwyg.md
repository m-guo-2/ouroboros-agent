# Agent 技能绑定所见即所得修复

- **日期**：2026-03-10
- **类型**：Bug 修复
- **状态**：已实施

## 背景

Admin 中关闭 Agent 的技能绑定后，刷新页面仍会看到技能像是还绑定着。根因有两个：前端把技能 `name` 当成绑定键，而后端和数据库实际存的是 `id`；同时后端把空 skills 解释成“加载全部启用技能”。

## 决策

将 Agent 技能绑定统一收敛到技能 `id`，并把空 skills 语义改成“明确不绑定任何技能”。

## 变更内容

- `admin/src/components/features/agents/agent-detail.tsx`：技能开关改为基于 `skill.id` 读写，避免 `id/name` 不一致导致的显示与持久化错位。
- `admin/src/hooks/use-agents.ts`：更新 Agent 后同时失效列表和当前详情缓存，减少保存后的界面残留旧值。
- `agent/internal/storage/skills.go`：移除“空 skills 回退为全量启用”的逻辑，让空绑定在运行时也保持为空，只保留按需加载技能索引。

## 考虑过的替代方案

继续保留后端的空数组回退语义，只在前端上做兼容映射。该方案会让界面状态和运行时行为继续分裂，无法做到真正的所见即所得，因此放弃。

## 影响

今后 Agent 的 `skills` 字段语义更直接：存什么就加载什么；存空数组就是不默认加载任何技能。历史上依赖“空即全量”的 Agent，如需默认能力，必须显式写入对应 skill id。
