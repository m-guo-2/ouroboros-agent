# Main 分支显式直推放开

- **日期**：2026-03-06
- **类型**：讨论结论
- **状态**：已决定

## 背景

仓库工作流文档此前要求一律不要直接推送 `main`。在实际使用中，存在用户已经明确知晓风险、并要求 agent 直接把本地已提交 commit 推送到 `main` 的场景。

## 决策

保留“分支 + PR”作为默认推荐流程，但当用户明确要求时，允许 agent 直接推送 `main`。

## 变更内容

- 更新 `CLAUDE.md` 中的 Git Workflow 描述
- 将“禁止直推 main”调整为“默认不直推，但用户显式要求时允许”
- 保留推荐工作流，避免默认协作模式被完全放松

## 考虑过的替代方案

继续保持完全禁止直推 `main`。该方案更保守，但无法满足仓库 owner 在特定场景下的直接发布诉求。

## 影响

后续 agent 在处理 push 请求时，可以区分“默认建议”与“用户明确授权”。常规开发仍应优先走分支和 PR，只有在用户明确要求时才直接推送主分支。
