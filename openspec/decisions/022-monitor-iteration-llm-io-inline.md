# Monitor 按轮次内联展示 LLM I/O

- **日期**：2026-02-28
- **类型**：代码变更
- **状态**：已实施

## 背景

当前 Monitor 已支持通过 `llmIORef` 查看完整模型请求/响应，但入口是独立按钮 + 弹窗。对于排查多轮 ReAct 过程，切换弹窗会打断阅读上下文，不利于逐轮对照“思考/工具调用/模型输入输出”。

## 决策

在每个 Iteration 区块内新增可折叠的内联 `Model I/O` 面板，按需加载该轮 `request/response`，让用户在同一视图中完成逐轮排查。

## 变更内容

- 前端 `monitor-page.tsx` 新增 `LLMIOInline` 组件：
  - 使用 `traceId + llmIORef` 发起查询；
  - 默认折叠，展开时才请求数据（`enabled: expanded`）；
  - 分区展示 `Input`（request）和 `Output`（response）。
- 在 `IterationGroup` 的展开内容顶部插入内联 I/O 区块：
  - 仅当该轮存在 `llmIORef` 时渲染；
  - 保留原有迭代统计和工具链展示逻辑。

## 考虑过的替代方案

- 仅保留弹窗查看：实现简单，但跨轮对照时频繁开关弹窗，认知负担较高。
- 默认自动展开所有 I/O：信息密度过高，长会话下会明显影响可读性与性能。

## 影响

- 用户可在同一轮次上下文内直接查看模型输入输出，排障效率提升。
- I/O 数据仍为按需加载，不会增加默认渲染开销。
- 与现有后端 `/api/traces/:traceId/llm-io/:ref` 协议保持兼容，无需后端改动。
