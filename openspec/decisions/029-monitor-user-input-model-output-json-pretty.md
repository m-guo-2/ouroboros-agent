# Monitor 补齐用户输入与模型输出主视图

- **日期**：2026-03-02
- **类型**：代码变更
- **状态**：已实施

## 背景

当前 Monitor 的执行过程虽然有轮次和工具调用，但主视角不够直观：用户输入在迭代视图中不突出，模型每轮输出不明显，JSON 字段可读性差。排障时需要频繁切换和手动解析，信息密度低。

## 决策

在不改业务逻辑的前提下，仅通过前端展示增强可观测性：每轮优先展示用户输入与模型输出，工具细节继续折叠；所有 JSON-like 字段统一做格式化展示。

## 变更内容

- 更新 `admin/src/components/features/monitor/monitor-page.tsx`：
  - 新增 `UserInputView`，在每轮视图中显式展示用户输入（首轮默认可见）。
  - 新增 `ModelOutputInline`，基于已有 `llmIORef` 按需加载并提取本轮模型输出文本。
  - 新增 `safePretty` / `PrettyPayload`，统一处理字符串与对象的 JSON 格式化展示。
  - 工具 `input/result` 渲染改为统一 pretty 渲染，减少原始字符串噪音。
- 保持现有业务代码与协议不变：未修改 agent/server 的执行流程、消息协议、数据库结构。

## 考虑过的替代方案

- 后端新增每轮 `content` 事件并直接返回 `iterationOutput`：可读性最好，但涉及业务侧事件模型改动，不符合本次“仅展示层改动”的约束。

## 影响

- 监控页默认信息更聚焦，排障路径从“看工具日志”转为“先看用户输入/模型输出，再下钻细节”。
- 性能上保持按需加载：模型输出依赖 `llmIORef`，仅在展开时请求与解析，避免初始渲染处理大 JSON。
