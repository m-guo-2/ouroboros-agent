# Admin 文本编辑区自动扩高

- **日期**：2026-03-04
- **类型**：代码变更
- **状态**：已实施

## 背景

admin 后台多个编辑场景（如系统提示词、README、JSON 配置）使用统一文本框组件，默认窗口高度偏小。内容稍长时需要频繁滚动，编辑效率和可读性较差。

## 决策

在 `admin` 的统一 `Textarea` 组件中内置按内容自动扩高逻辑，默认启用，并保留最大行数限制能力。

## 变更内容

- 更新 `admin/src/components/ui/textarea.tsx`：
  - 新增 `autoResize`（默认 `true`）与 `maxAutoResizeRows` 属性；
  - 在输入和内容变更时根据 `scrollHeight` 自动调整高度；
  - 当设置 `maxAutoResizeRows` 时，超过阈值改为内部滚动，避免无限拉长页面。
- 现有使用 `Textarea` 的后台页面（如 Agent/Skill 相关编辑器）无需逐个改造即可受益。

## 考虑过的替代方案

- 仅在个别页面手动调大 `rows`：实现简单，但无法随内容动态变化，且需要分散维护。
- 单独新增自动扩高组件并逐页替换：可控性高，但改造面更大，收益不及统一基座改造。

## 影响

- 长文本编辑体验明显改善，减少“看不全内容”的问题。
- 组件 API 向后兼容；如需恢复固定高度，可通过 `autoResize={false}` 控制。
