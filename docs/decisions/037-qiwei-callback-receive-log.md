# 企微回调接收日志补充

- **日期**：2026-03-04
- **类型**：代码变更
- **状态**：已实施

## 背景

`channel-qiwei` 的回调链路仅在失败时打印错误日志，成功接收回调时没有显式日志。联调阶段用户发出消息后，无法快速确认服务是否收到了回调请求。

## 决策

在回调入口增加一条轻量接收日志，优先解决“是否收到回调”的可观测性问题。

## 变更内容

- 修改 `channel-qiwei/events.go`
  - 在 `handleWebhookCallback` 完成 body 解析后，新增日志：
    - `qiwei callback received messages=%d`
  - 保持现有异步处理与错误日志逻辑不变

## 影响

- 企微消息联调时，可直接通过终端确认回调是否到达。
- 不改变回调处理时序、业务逻辑和返回结构，仅提升可观测性。
